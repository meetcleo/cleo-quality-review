# Implementation Plan: Code Quality Review Gem

**Branch**: `001-code-quality-review` | **Date**: 2026-05-18 | **Spec**: [spec.md](./spec.md)

## Summary

Align existing `cleo_quality_review` gem implementation with spec requirements. The gem is ~90% complete. Key gaps: add `--only` alias for `--checks`, add `--exclude` flag, add explicit `--changed` flag, and update spec to reflect actual CLI name (`check_quality` per constitution).

## Technical Context

**Language/Version**: Ruby >= 3.2.0

**Primary Dependencies**: fasterer, flog, reek (quality tools), minitest (testing)

**Storage**: Filesystem (`tmp/quality_checks/[timestamp]/`)

**Testing**: Minitest with `bundle exec rake test`

**Target Platform**: Unix/macOS/Windows (Ruby runtime)

**Project Type**: Library + CLI gem

**Performance Goals**: Single file review < 30 seconds

**Constraints**: Non-human formats must work without network/LLM access

**Scale/Scope**: Individual file review, not repository-wide analysis

## Implementation Tasks

### Task 1: Add `--only` as alias for `--checks`

**Files**: `lib/cleo_quality_review/options.rb`, `test/options_test.rb`

**Changes**:
- Add `--only` option that populates same `checks` field
- Update help text to document both options
- Add tests for `--only` flag

### Task 2: Add `--exclude` flag

**Files**: `lib/cleo_quality_review/options.rb`, `lib/cleo_quality_review/runner.rb`, `test/options_test.rb`, `test/runner_test.rb`

**Changes**:
- Add `--exclude` option to `ParseResult` struct
- Parse comma-separated or repeated values
- In `Runner`, filter out excluded checks from registry result
- Add tests for exclusion logic

### Task 3: Add `--changed` flag

**Files**: `lib/cleo_quality_review/options.rb`, `lib/cleo_quality_review/target_resolver.rb`, `test/options_test.rb`, `test/target_resolver_test.rb`

**Changes**:
- Add `--changed` boolean flag to `ParseResult`
- When explicit files provided AND `--changed` not set, use explicit files only
- When `--changed` set (or no files provided), use git diff behavior
- Add tests for explicit vs changed mode

### Task 4: Update spec to match implementation

**Files**: `specs/001-code-quality-review/spec.md`

**Changes**:
- Replace `quality_review` with `check_quality` throughout
- Note: This aligns spec with constitution requirement
